Chapter 11 Notes

Key Terms

  • Custom Properties - You can add a custom propterty with dot notation
    • Example: myFunction.description = 'This is my description'
  • Temporary variables - Variable can be instantiated inside an immediately-invoked function express and won't be available afterwards
  • Strict Mode - Forces the browser to not allow sloppy coding practices and has to be strictly written
    • At top of JS file write 'use strict'
  • Recursive function - A function can call itself until a certain condition is met
  • Promise - Represents the future result of an asynchronous operation. Can be done with callbacks but are simpler
    • Example:
      const promise = new Promise( (resolve, reject) => {
      // initialization code goes here
      if (success) {
      resolve(value);
      } else {
      reject(error);
      } });
    • After promise has been settled, you use then() to deal with the outcome
    • catch() can be used with then() to deal with the failed state
  • Pure function
    1. Return value only depends on argument values
    2. No side-effects. It doesn't change any other data in the program
    3. Referential transparency - with saem arguments, it will always return the same result

Chapter 13 Notes

  • AJAX - Asynchronous JavaScript XML
  • fetch() - returns a promise that resolves to the response return from the URL
    • Example for getting data from an api
      const url = 'https:example.com/data';
      fetch(url)
      .then((response) => {
      if(response.ok) {
      return response;
      }
      throw Error(response.statusText);
      })
      .then( response => // do something with response )
      .catch( error => console.log('There was an error!') )
  • Redirect - redirects to another URL
  • blob() - reads file of raw data such as an image or spreadsheet
  • json() - turns JSON data into a JavaScript object to make it easier to deal with
